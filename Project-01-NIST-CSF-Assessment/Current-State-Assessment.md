# Enterprise Cybersecurity Assessment

## Current State Assessment

Project Number: NHS-CSF-001

Framework: NIST Cybersecurity Framework (CSF) 2.0

Assessment Owner: Carmelo Morgan

Assessment Status: Completed

-----

## Executive Summary

Northwind Health Solutions maintains a developing cybersecurity program supported by established technical controls, a growing governance structure, and executive recognition of cybersecurity as a business priority.

The organization has implemented foundational security controls including multi-factor authentication, endpoint protection, centralized identity management, vulnerability scanning, security awareness training, and routine data backups. These controls provide a solid baseline for protecting critical business systems and patient information.

While the technical security program demonstrates moderate maturity, several governance and operational processes remain informal or inconsistently documented. Opportunities exist to improve enterprise risk management, policy governance, control documentation, vendor oversight, security metrics, and formal remediation tracking.

Overall, Northwind Health Solutions demonstrates a cybersecurity maturity level consistent with an organization transitioning from reactive security management toward a structured, risk-based cybersecurity program.

## Function Assessments

## Govern (GV)

Current Maturity: Level 2 – Developing

### Strengths

* Executive leadership recognizes cybersecurity as a business priority.
* Core security policies have been established.
* Security responsibilities are assigned within the IT department.
* Annual cybersecurity planning is performed.

### Opportunities for Improvement

* Formal governance processes require additional documentation.
* Enterprise risk management should be more closely integrated with strategic planning.
* Security metrics should be consistently reported to leadership.
* Third-party risk management processes should be standardized.
* Policy review schedules should be formally documented.

### Overall Assessment

Governance activities are present but remain largely operational rather than strategic. Additional formalization would improve consistency, accountability, and executive visibility.

-----

## Identify (ID)

Current Maturity: Level 3 – Defined

### Strengths

* Hardware and software inventories are maintained.
* Critical business systems have been identified.
* Data classification standards have been established.
* Cloud assets are documented.
* Business-critical services have been prioritized.

### Opportunities for Improvement

* Asset ownership should be reviewed periodically.
* Automated asset discovery should be expanded.
* Business impact analyses should be updated annually.
* Vendor inventories should be maintained within a centralized repository.

### Overall Assessment

Asset management practices provide an effective foundation for risk management, although additional automation and governance would improve accuracy.

-----

## Protect (PR)

Current Maturity: Level 3 – Defined

### Strengths

* Multi-factor authentication is enabled for privileged accounts.
* Role-based access control has been implemented.
* Endpoint protection is deployed across managed systems.
* Password standards are documented.
* Security awareness training is conducted annually.
* Backup processes are established.

### Opportunities for Improvement

* Privileged access reviews should occur more frequently.
* Security awareness metrics should be measured.
* Data loss prevention capabilities should be expanded.
* Secure configuration baselines should be documented.

### Overall Assessment

Protective controls are generally effective and consistently implemented across the organization.

-----

## Detect (DE)

Current Maturity: Level 2 – Developing

### Strengths

* Centralized log collection has been implemented.
* Endpoint security alerts are monitored.
* Vulnerability scans are performed regularly.
* Cloud logging has been enabled.

### Opportunities for Improvement

* Security monitoring procedures should be documented.
* Detection use cases should be expanded.
* Alert prioritization should be standardized.
* Security metrics should be routinely reviewed.

### Overall Assessment

Detection capabilities provide visibility into common security events but require additional process maturity to improve consistency and response effectiveness.

-----

## Respond (RS)

Current Maturity: Level 2 – Developing

### Strengths

* An incident response plan has been developed.
* Security incidents are documented.
* Roles and responsibilities have been identified.

### Opportunities for Improvement

* Tabletop exercises should be conducted annually.
* Lessons learned should be formally documented.
* Communication procedures should be standardized.
* Incident severity classifications should be defined.
* Post-incident reviews should be integrated into governance activities.

### Overall Assessment

Incident response processes exist but are primarily reactive. Additional planning, testing, and documentation would improve organizational resilience.

-----

## Recover (RC)

Current Maturity: Level 3 – Defined

### Strengths

* Backup procedures are documented.
* Disaster recovery procedures have been established.
* Recovery priorities have been identified.
* Critical business systems have documented recovery objectives.

### Opportunities for Improvement

* Recovery exercises should be performed more frequently.
* Recovery metrics should be reported to leadership.
* Business continuity documentation should be reviewed annually.
* Recovery procedures should include cloud-specific scenarios.

### Overall Assessment

Recovery planning is well established and provides confidence in the organization's ability to restore critical operations following a disruptive event.

## Organizational Strengths

* Strong executive support for cybersecurity initiatives.
* Established identity and access management practices.
* Effective endpoint security deployment.
* Comprehensive backup and recovery capabilities.
* Documented asset management practices.

## Overall Conclusion

Northwind Health Solutions demonstrates a stable cybersecurity foundation supported by effective technical safeguards and growing organizational commitment to cybersecurity.

The organization is well positioned to advance from a developing cybersecurity program toward a mature, risk-driven security governance model.

The recommendations presented in subsequent deliverables focus on strengthening governance processes, improving operational consistency, and increasing organizational resilience while leveraging the existing technical capabilities already in place.
