## Purpose

This document describes the methodology used to assess the cybersecurity posture of Northwind Health Solutions. The assessment follows the National Institute of Standards and Technology (NIST) Cybersecurity Framework (CSF) 2.0 and is intended to provide a structured, repeatable process for evaluating the effectiveness of the organization's cybersecurity program.

The methodology emphasizes risk-based decision making and focuses on evaluating governance, technical safeguards, operational processes, and organizational resilience.

-----

## Assessment Goals

The assessment was designed to achieve the following objectives:

* Establish a baseline cybersecurity maturity.
* Evaluate existing security controls.
* Identify control gaps.
* Assess organizational risk.
* Prioritize remediation activities.
* Improve alignment with industry best practices.
* Support long-term cybersecurity planning.

## Govern (GV)

## Evidence Sources

The assessment considers multiple sources of evidence to evaluate the effectiveness of organizational security controls.

Evidence includes:

* Security policies
* Technical configurations
* Existing procedures
* Asset inventories
* Access control documentation
* Vulnerability assessment results
* Incident reports
* Risk registers
* System architecture documentation
* Security awareness records
* Cloud configuration reviews
* Administrative settings
* Security dashboards
* Internal documentation

No single source of evidence is considered sufficient on its own. Findings are based on the overall effectiveness of multiple control areas.

## Assessment Process

The assessment follows seven phases.

## Phase 1 – Understand the Environment

Review organizational documentation to understand:

* Business operations
* Technology environment
* Critical assets
* Regulatory obligations
* Existing security program

Deliverables reviewed include:

* Enterprise Architecture
* Network Diagram
* Asset Inventory
* Technology Stack
* Security Baseline

-----

## Phase 2 – Review Existing Controls

Evaluate existing administrative, technical, and operational controls.

Examples include:

* Password policies
* Multi-factor authentication
* Logging
* Endpoint protection
* Backup procedures
* Security awareness training
* Vulnerability management
* Incident response processes

-----

## Phase 3 – Evaluate Control Effectiveness

Each control is evaluated based on:

* Design
* Implementation
* Operational effectiveness
* Documentation
* Consistency

Controls are not considered effective solely because they exist. Controls must also demonstrate evidence of consistent implementation and support organizational objectives.

-----

## Phase 4 – Identify Gaps

Compare the current security posture against the expectations of the NIST Cybersecurity Framework.

Each identified gap is documented along with:

* Description
* Associated risk
* Potential business impact
* Recommended remediation

-----

## Phase 5 – Risk Evaluation

Every identified gap is evaluated using a qualitative risk assessment.

Risk considers:

* Likelihood
* Business impact
* Existing controls
* Residual risk

Risk ratings include:

* Critical
* High
* Moderate
* Low

-----

## Phase 6 – Develop Recommendations

Recommendations are developed according to the following principles:

* Reduce organizational risk
* Improve security maturity
* Support business objectives
* Balance security with operational requirements
* Prioritize high-impact improvements

Each recommendation includes:

* Description
* Business justification
* Estimated implementation effort
* Relative priority

-----

## Phase 7 – Develop Roadmap

Recommendations are organized into an implementation roadmap using three planning horizons.

### Immediate (0–90 Days)

High-priority activities that reduce significant organizational risk.

### Near-Term (3–6 Months)

Projects requiring planning, coordination, or moderate implementation effort.

### Long-Term (6–12 Months)

Strategic initiatives focused on improving organizational maturity and long-term resilience.

-----

## Maturity Scoring Model

Each NIST CSF function is evaluated using the following maturity scale.

| Level                | Description                                                                                                |
| -------------------- | ---------------------------------------------------------------------------------------------------------- |
| Level 1 – Initial    | Processes are informal, inconsistent, or reactive. Documentation is minimal or absent.                     |
| Level 2 – Developing | Basic controls exist but are inconsistently implemented or only partially documented.                      |
| Level 3 – Defined    | Policies, procedures, and controls are formally documented and generally followed across the organization. |
| Level 4 – Managed    | Controls are consistently implemented, measured, and regularly reviewed for effectiveness.                 |
| Level 5 – Optimized  | Security practices are continuously improved through metrics, automation, and organizational learning.     |

Maturity scores represent the current state of organizational capability rather than compliance status.

-----

## Assessment Principles

This assessment follows several guiding principles.

* Security is aligned with business objectives.
* Risk management drives prioritization.
* Evidence is preferred over assumptions.
* Recommendations should be practical and achievable.
* Continuous improvement is emphasized over perfect compliance.
* Security maturity is viewed as an ongoing process rather than a final state.

-----

## Assessment Limitations

The assessment is intended to provide a high-level evaluation of organizational cybersecurity maturity.

It does not replace:

* Formal compliance audits
* Penetration testing
* Red team exercises
* Vulnerability scanning
* Digital forensic investigations

These activities should be performed separately as part of the organization's broader cybersecurity program.
