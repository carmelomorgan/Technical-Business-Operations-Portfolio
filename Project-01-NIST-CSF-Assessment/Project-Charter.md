## Project Purpose

The purpose of this project is to evaluate the current cybersecurity posture of Northwind Health Solutions using the National Institute of Standards and Technology (NIST) Cybersecurity Framework (CSF) 2.0.

This assessment establishes a baseline understanding of the organization's cybersecurity maturity, identifies strengths and areas for improvement, evaluates existing security controls, and provides actionable recommendations for reducing organizational risk.

The assessment is intended to support informed decision-making by leadership while helping prioritize future cybersecurity initiatives based on business risk.

-----

## Background

Northwind Health Solutions is a regional healthcare provider responsible for protecting sensitive patient information, business data, and critical healthcare operations. The organization relies on a hybrid technology environment consisting of on-premises infrastructure, Microsoft 365 services, Active Directory, Linux servers, and Amazon Web Services (AWS).

As the organization's cybersecurity program continues to mature, leadership has requested a comprehensive assessment of existing security controls to determine alignment with recognized industry best practices and identify opportunities for improvement.

-----

## Project Objectives

The objectives of this assessment are to:

* Establish a baseline of the organization's current cybersecurity maturity.
* Evaluate existing administrative, technical, and operational security controls.
* Identify gaps in the organization's cybersecurity program.
* Assess risks associated with identified control deficiencies.
* Prioritize remediation efforts based on business impact and organizational risk.
* Improve alignment with the NIST Cybersecurity Framework.
* Support future policy development and security program improvements.

-----

### Excluded

The following activities are outside the scope of this assessment:

* Penetration testing
* Red team exercises
* Source code review
* Physical security testing
* Social engineering campaigns
* Regulatory certification audits
* Production system modifications

## Assessment Framework

This project uses the NIST Cybersecurity Framework (CSF) 2.0 as the primary assessment methodology.

The assessment evaluates organizational capabilities across the following functional areas:

* Govern
* Identify
* Protect
* Detect
* Respond
* Recover

Each function will be assessed using available organizational documentation, technical configurations, existing security controls, and operational processes.

-----

## Expected Deliverables

This project will produce the following documentation:

* Assessment Methodology
* Current State Assessment
* Gap Analysis
* Enterprise Risk Analysis
* Security Recommendations
* Implementation Roadmap
* Executive Summary

Each deliverable is intended to support future security initiatives and provide a repeatable methodology for assessing cybersecurity maturity over time.

-----

## Success Criteria

The project will be considered successful if it:

* Establishes an accurate baseline of the current cybersecurity program.
* Identifies significant security gaps and associated risks.
* Provides practical, risk-based recommendations for improvement.
* Produces documentation suitable for executive review.
* Creates reusable artifacts that support ongoing governance, risk management, and continuous improvement efforts.

-----

## Assumptions

The assessment is based on the following assumptions:

* Organizational leadership supports continuous improvement of the cybersecurity program.
* Existing documentation accurately reflects current operational practices unless otherwise noted.
* Technical evidence reviewed during the assessment is representative of the production environment.
* Recommendations will be prioritized according to organizational risk tolerance, available resources, and business impact.

-----

## Related Portfolio Projects

This assessment serves as the foundational project for the Northwind Health Solutions cybersecurity portfolio.

Subsequent projects—including the Enterprise Risk Register, Security Policy Library, Vulnerability Assessment, Access Control Review, Security Metrics Dashboard, and Cloud Security Assessment—will reference the findings and recommendations established through this assessment.
